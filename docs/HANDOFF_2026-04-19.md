## 3. Strategic Plans (pilih 1 untuk sesi berikut)

**Tujuan:** extend Threads social agent ke X/LinkedIn/Reddit/Discord/Telegram/YouTube/Medium supaya SIDIX bisa dipromosikan di semua channel.
**File kunci:** `apps/brain_qa/brain_qa/channel_adapters.py`, `social_agent.py`, `threads_autopost.py`.
**Langkah:**

2. Buat `connectors/linkedin_connector.py` (OAuth + UGC posts)
3. Buat `connectors/reddit_connector.py` (PRAW atau HTTP langsung)

5. Tambah cron `0 10 * * * /opt/sidix/scripts/cross_post.sh`
**Output:** 1 tulisan → 7 channel. KPI: followers bertambah cross-platform.

## 4.5. Arsitektur SIDIX (penting untuk agent sesi berikut)

SIDIX **BUKAN** RAG/retrieval sederhana. 3-layer design (LOCK, lihat `CLAUDE.md` section IDENTITAS SIDIX):

1. **LLM generative core** — Qwen2.5-7B + LoRA SIDIX fine-tune own. Generate jawaban via token prediction. Tetap jawab walau corpus kosong.
2. **RAG + agent tools** (17 aktif) — sensory: corpus (BM25), web (fetch+search), komputasi (code_sandbox), file (pdf_extract, workspace). ReAct loop bikin SIDIX jadi **agent**, bukan chatbot.
3. **Growth loop autonomous** — LearnAgent → corpus → daily_growth → corpus_to_training → auto_lora retrain. Model makin pintar tiap kuartal.

Implikasi untuk plan berikut:
- Plan apapun WAJIB pertahankan ketiga layer jalan.
- Tambah tool = augment agent, tidak ganti LLM.
- Tambah data = feed growth loop supaya LoRA retrain.

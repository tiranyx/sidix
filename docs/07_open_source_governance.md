# Open-source Governance (draft)

## Lisensi
- MIT (lihat `LICENSE`).

## Struktur kontribusi
- Issue untuk bug/feature proposal.
- PR wajib menyertakan:
  - ringkasan perubahan,
  - test plan (manual/otomatis),
  - dampak backward compatibility.

## RFC (opsional untuk perubahan besar)
Perubahan besar (auth, plugin system, data model) sebaiknya lewat dokumen RFC singkat di `docs/rfcs/`.

## Code of Conduct
Tambah `CODE_OF_CONDUCT.md` (Contributor Covenant) saat repo dipublish.

## Maintainers
Mulai dari 1–2 maintainer. Tambah co-maintainer saat kontribusi stabil.

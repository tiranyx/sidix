# Batasan & red flags (versi awal)

## Hal yang biasanya tidak saya mau
- Janji berlebihan seperti “self-improving tanpa batas”.
- Saran yang butuh biaya besar tanpa estimasi biaya/risiko.
- Mengubah banyak hal sekaligus tanpa rencana bertahap.

## Red flags keamanan/privasi
- Memasukkan data sensitif ke repo publik.
- Menjalankan tool/command yang destruktif tanpa izin jelas.
- Menyimpan API keys di file yang akan di-commit.

## Prinsip open-source
- Default: publik hanya yang aman & punya hak distribusi.
- Private data harus masuk `brain/private/` (di-ignore).
